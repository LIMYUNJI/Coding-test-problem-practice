컴포지션 <========> 내부 클래스
컴포지션: 내부 객체를 통해서 원본 객체의 기능을 사용
- 클래스들 간에 관계를 맺을 경우에는 상속을 썼으나(확장 개념) 요즘에는 클래스 내부에 선언 하는 것이 좋다.


내부 클래스 종류

1. 인스턴스 내부 클래스
2. 정적(static) 내부 클래스
3. 로컬 클래스
4. 익명 내부 클래스

내부 클래스 사용 장점:

접근 제어 지시자
public : 어디서든 자유롭게 접근 가은
protected : 같은 패키지에서는 자유롭게 접근가능, 자식 클래스 라면(상속받은 경우) 다른 패키지에서 접근 가응
private : 같은 클래스만 접근 가능, 상속 받은 경우에는 접근 불가능
default : 같은 패키지 내에서 접근 가능
