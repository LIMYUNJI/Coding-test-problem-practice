소프트웨어 설계
현행 시슽템 분석에는 플랫폼 분석 기능, 플랫폼 성능 특성 분석, 운영 체게 분석, DBMS분석, 운영체제 분석, 비즈니스 융합 분석이 있다.

멀티 버스 활성화를 위한 CPND 의미
C : 콘텐츠 P : 플랫폼 N : 네트워크 D : 디바이스

플랫폼이란 수많은 애플리케이션을 실행하는데 상호호환이 쉽게 이루어질 수 있도록 만들어진 환경을 의미한다

플랫폼의 기능
1. 연결기능 2. 비용감소 기능 3. 검색 비용 절약 기능 4. 커뮤니티 형성에 의한 네트워크 효과

운영체제 
하드웨어와 소프트 웨어 자원을 관리하고 컴퓨터의 공통 서비스를 제공하는 소프트웨어를 말한다.

운영체제의 종류
1. 윈도우 - 일반 ㅖC 유지등 관리 비용 장점
2. 유닉스 - 대용량 처리, 안전성 높은 엩터프라이즈 급 서버
3. 리눅스 - 중/대규모 서버 높은 보안성 제공
4. IOS - 다양한 기기와의 홓 ㅘㄴ성 제공
5. 안드로이드 - 높은 보안성과 고성능 제공

네크트워크
동일한 프로토콜을 사용하는 장치들의 집합

OSI 7게층

물리적 계층 - 데이터 계층 - 네트워크 계층 - 전송 계층 - 세션 계층 - 표층 계층 - 응용계층

소프트웨어 설계
요구사항을 기반으로 시스템 구조와 동작을 정의하는 단계. 오류 최소화, 유지보수 용이, 품질 향상 목적.

설계 원리
추상화: 핵심만 표현(자료/제어/기능)
모듈화: 기능 단위 분리
단계적 분해: 상위 기능 → 하위 기능
정보 은닉: 내부 구현 숨기고 인터페이스만 공개

모듈 설계
결합도(Coupling): 모듈 간 의존도, 낮을수록 좋음
내용 > 공통 > 외부 > 제어 > 스탬프 > 자료(최상)
응집도(Cohesion): 모듈 내부 관련성, 높을수록 좋음
우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적(최상)

소프트웨어 아키텍처
시스템 구성요소와 관계 정의
스타일: 계층형, 클라이언트-서버, 파이프-필터, MVC

UML
객체지향 모델링 표준
구조 다이어그램: 클래스, 객체, 컴포넌트, 배치, 패키지
행위 다이어그램: 유스케이스, 시퀀스, 상태, 활동
유스케이스: 사용자 관점 기능 표현

인터페이스 설계
시스템/모듈 간 연결 지점
원칙: 명확성, 일관성, 단순성, 변경 최소화

사용자 인터페이스(UI)
원칙: 직관성, 일관성, 학습성, 유연성

설계 산출물
요구사항 명세서, 설계 명세서, UML 다이어그램, 인터페이스 명세서

[암기 포인트]
결합도 ↓ 응집도 ↑
자료 결합도 / 기능적 응집도
UML = 구조 + 행위
유스케이스 = 사용자 관점
